# Динамическая фильтрация и рейтинг серверов

## 🎯 Концепция

Xpert Panel теперь использует **умную систему рейтинга** - в подписку попадают только топ-10 лучших серверов, остальные ждут своей очереди.

## 📋 Как это работает

### **1. Система рейтинга (Score 0-100)**
- **Success Rate** (40 баллов) - процент успешных пингов
- **Average Ping** (35 баллов) - средний пинг (меньше = лучше)
- **Unique Users** (25 баллов) - количество пользователей (больше = надежнее)
- **Оригинальные метрики** - если нет статистики пользователей

### **2. Фильтрация и очередь**
- **Топ-N серверов** попадают в подписку (по умолчанию 10)
- **Остальные серверы** находятся в очереди
- **Динамическое обновление** - рейтинг пересчитывается при каждом запросе

### **3. Двойная фильтрация**
1. **Health Filter** - только здоровые серверы (≥70% success, ≤1000ms ping)
2. **Top Filter** - только топ-N серверов по рейтингу

## 🛠 API для управления рейтингом

### **GET /api/xpert/top-configs?limit=10**
Получение топ-N серверов с рейтингом:

```json
{
  "configs": [
    {
      "id": 1,
      "server": "example.com",
      "port": 443,
      "protocol": "vless",
      "ping_ms": 145.2,
      "score": 87.5,
      "health": {
        "healthy": true,
        "avg_ping": 145.2,
        "success_rate": 85.5,
        "unique_users": 8
      }
    }
  ],
  "total": 10
}
```

### **GET /api/xpert/queue-configs**
Получение серверов в очереди:

```json
{
  "configs": [...],
  "total": 25
}
```

## ⚙️ Настройки

```bash
# Количество топ серверов в подписке
XPERT_TOP_SERVERS_LIMIT=10

# Включить динамическую фильтрацию
XPERT_USE_DYNAMIC_FILTERING=true

# Минимальное количество пользователей для статистики
XPERT_MIN_USERS_FOR_STATS=3
```

## 🔄 Расчет рейтинга

### **При наличии статистики пользователей:**
```
Score = Success Rate * 0.4 + Ping Score * 0.35 + Users Score * 0.25

Success Rate: 85% → 34 балла (85 * 0.4)
Ping Score: 150ms → 32.5 балла ((1000-150)/1000 * 35)
Users Score: 8 пользователей → 20 баллов (min(8, 25))
Total Score: 86.5 баллов
```

### **При отсутствии статистики:**
```
Score = Ping Score * 0.5 + Loss Score * 0.3 + Active Bonus * 0.2

Ping Score: 150ms → 42.5 балла ((1000-150)/1000 * 50)
Loss Score: 5% → 28.5 баллов ((100-5)/100 * 30)
Active Bonus: 20 баллов
Total Score: 91 балл
```

## 📊 Workflow

1. **Проверка здоровья** - отсеиваем нерабочие серверы
2. **Расчет рейтинга** - оцениваем каждый сервер
3. **Сортировка** - по убыванию рейтинга
4. **Отбор топ-N** - для подписки
5. **Очередь** - остальные ждут повышения рейтинга

## 🎯 Преимущества

✅ **Только лучшие** - в подписке попадают топ серверы  
✅ **Мотивация** - серверы борются за место в топе  
✅ **Адаптивность** - рейтинг меняется в реальном времени  
✅ **Контроль** - легко настроить количество серверов  

## � Как сервер попадает в топ

### **Пути улучшения рейтинга:**
1. **Улучшить пинг** - техническая оптимизация
2. **Повысить надежность** - стабильность работы
3. **Привлечь пользователей** - больше пингов = точнее статистика

### **Причины падения рейтинга:**
1. **Высокий пинг** - медленные соединения
2. **Потери пакетов** - нестабильность
3. **Мало пользователей** - недостаточно данных

### **Текущий топ-10:**
```bash
curl https://your-domain.com/api/xpert/top-configs
```

## 📝 Пример использования

### **Администратор:**
- Настроил `XPERT_TOP_SERVERS_LIMIT=15`
- Следит за топом через API
- Анализирует очередь серверов

### **Пользователь:**
- Получает только 15 лучших серверов
- Клиент пингует и отправляет статистику
- Влияет на рейтинг серверов

### **Владелец сервера:**
- Следит за своим рейтингом
- Оптимизирует производительность
- Привлекает пользователей для улучшения статистики
