## 🚀 Как запустить

### 1. Перезапустите Xpert Panel
```bash
# Если запущено как сервис
sudo systemctl restart xpert

# Если запущено вручную
# Остановите Ctrl+C и запустите заново
python3 main.py
```

### 2. Откройте UI
```
http://your-domain.com
```

### 3. Перейдите к Traffic Manager
```
1. Нажмите на меню ☰ (три полоски вверху)
2. Выберите "Traffic Manager"
Или перейдите напрямую: http://your-domain.com/#/traffic/
```

## 🔗 Интеграция с Xpert

### Кнопки в Xpert UI теперь работают с внешним трафиком:
- ✅ **"Сбросить трафик"** - очищает Xpert + внешний трафик
- ✅ **"Лимит трафика"** - учитывает весь трафик в лимитах

### API эндпоинты доступны:
```
GET  /api/admin/usage/{username}              - Общее использование
GET  /api/admin/usage/{username}/detailed   - Детальная статистика
POST /api/admin/usage/reset/{username}       - Сброс всего трафика
```

## 🧪 Тестирование UI

### Проверка базового функционала
1. Откройте Traffic Manager
2. Убедитесь что статистика загружается
3. Проверьте глобальные цифры
4. Введите user_token и посмотрите детальную статистику

### Тестирование управления
1. Нажмите "Refresh" - статистика должна обновиться
2. Нажмите "Reset All" - подтвердите сброс
3. Проверьте что статистика обнулилась

### Тестирование с реальным трафиком
1. Используйте подписку с `user_token=test_user`
2. Отправьте webhook данные:
```bash
curl -X POST "http://your-domain.com/api/xpert/traffic-webhook" \
  -H "Content-Type: application/json" \
  -d '{
    "user_token": "test_user",
    "server": "vpn.example.com",
    "port": 443,
    "protocol": "vless",
    "bytes_uploaded": 1048576,
    "bytes_downloaded": 2097152
  }'
```
3. Обновите UI - должна появиться новая статистика

## 🛠️ Конфигурация

### Переменные окружения (.env)
```env
# Включить систему мониторинга
XPERT_TRAFFIC_TRACKING_ENABLED=True

# Путь к базе данных
XPERT_TRAFFIC_DB_PATH=data/traffic_stats.db

# Хранение данных (0 = бесконечно)
XPERT_TRAFFIC_RETENTION_DAYS=0

# Домен для webhook
XPERT_DOMAIN=your-domain.com
```

### База данных
```
Файл: data/traffic_stats.db
Таблица: traffic_usage
Индексы: user_token, server_port, timestamp
Размер: ~0.1MB на 1000 записей
```
